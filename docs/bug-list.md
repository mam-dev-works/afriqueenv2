# Client Bug List Index

Source document: `List of bugs.docx.pdf` also `List of bugs.md` in the project root.

Use this file to track and prioritise bugs while you work through the client document.

## Legend

- **P0** – App-breaking (crash, cannot login, cannot continue a core flow).
- **P1** – Major functional bug (wrong data, wrong flow, navigation failure).
- **P2** – Minor logic or display issue (text wrong, non-critical misbehaviour).
- **P3** – Pure cosmetic (spacing, colours, small UI polish).

### How to use

1. Go through `List of bugs.docx.pdf` page by page.
2. For each bug, add a new row with:
   - **Bug ID**: incrementing (e.g. `BUG-004`).
   - **Doc Page**: page number from the PDF.
   - **Area / Screen**: e.g. `Login`, `Profile`, `Home`, `Cart`.
   - **Title**: one-line summary in your own words.
   - **Priority**: choose P0–P3 based on impact.
   - **Status**: e.g. `Pending`, `In progress`, `Fixed`, `Cannot reproduce`.
   - **Notes**: short explanation of cause/fix once you understand it.

Keep this file up to date as you fix bugs so you can quickly report progress to the client.
