## Architecture Pattern: BLoC (Business Logic Component)

Each feature follows a consistent structure:
- **model/**: Data models and entities
- **repository/**: Data layer (Firebase, API calls)
- **bloc/**: Business logic (BLoC pattern with events, states, and bloc)
- **screen/**: UI screens
- **widget/**: Feature-specific reusable widgets

## Key Technologies
- State Management: flutter_bloc ^9.1.1
- Backend: Firebase (Auth, Firestore, Storage, Realtime DB, Dynamic Links)
- Image Management: Cloudinary
- Local Storage: get_storage, shared_preferences
- Localization: Multi-language support (EN, FR)

## Total Features: 30
Authentication, Profile Management, Chat, Stories, Matching, Events, Gifts, 
Settings, Activity Tracking, Privacy Controls, and more.
